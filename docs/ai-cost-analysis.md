> 다만 Gemini가 한국어 output 토큰 효율이 약간 높은 경향이 있어 실사용 비용은 Gemini가 ~10% 저렴할 수 있음.

## 참고사항

1. **Output은 maxTokens 기준 최대치** — 실제 응답은 보통 60~80% 사용. 실비용은 위 표의 70% 수준.
2. **상담소 대화 누적** — 10턴 이상 대화 시 input이 증가하나 MAX_HISTORY_TURNS=10으로 상한 제한.
3. **Gemini thinking OFF** — `thinkingBudget: 0` 설정으로 thinking 토큰 비용 없음. ON 시 output 비용 ~6배 증가.
4. **캐싱** — Gemini context caching 적용 시 input 비용 75% 절감 가능 (동일 시스템 프롬프트 반복 호출).
5. **2-pass 서비스** (신년/정통사주) — 2차에서 1차 결과를 context로 포함하므로 input이 상당히 증가. 비용의 주된 요인.
