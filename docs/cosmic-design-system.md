# 사주 코스믹 디자인 시스템 (Cosmic Design System)

> "우주의 기운을 드립니다"
> 사주명리학의 동양 철학적 깊이와 우주의 신비로움을 결합한 디자인 언어

---

## 0. 디자인 철학: 천문사주 (天文四柱)

### 핵심 메타포

사주명리학은 본래 천문(天文)에서 출발했다. 고대 동양에서 하늘의 별과 행성의 움직임을 관찰하여 인간의 운명을 읽었다. 이 디자인 시스템은 그 원래의 연결 고리를 현대적으로 복원한다.

| 사주 개념 | 우주적 재해석 | 디자인 표현 |
|-----------|-------------|------------|
| **사주(四柱)** | 4개의 항성 기둥 | 각 기둥이 빛나는 별처럼 표현, 수직으로 배열된 광주(光柱) |
| **오행(五行)** | 5종류의 성운(星雲) | 각 오행이 고유한 색상의 성운/가스 구름으로 표현 |
| **천간(天干)** | 항성(恒星) | 밝고 선명한 빛, 위쪽에 배치 |
| **지지(地支)** | 행성(行星) | 은은하고 무게감 있는 빛, 아래쪽에 배치 |
| **대운(大運)** | 공전 궤도 | 원형/타원형 궤도 위를 이동하는 빛의 점 |
| **격국(格局)** | 별자리 패턴 | 사주 기둥 4개를 잇는 선이 만드는 고유 형상 |
| **음양(陰陽)** | 밝은 면/그림자 | 광원과 그림자의 대비, 빛의 강도 차이 |
| **십신(十神)** | 위성/소행성 | 메인 기둥 주변을 도는 작은 빛 |
| **운세 흐름** | 혜성의 궤적 | 시간축을 따라 흐르는 빛의 트레일 |
| **해(크레딧)** | 태양 | 금빛 발광체, 코로나 효과 |
| **달(크레딧)** | 달 | 은빛 발광체, 부드러운 후광 |

### 디자인 원칙

1. **심원(深遠)** - 깊고 먼 느낌. 배경은 항상 우주의 깊이감을 가진다.
2. **발광(發光)** - 중요한 요소는 스스로 빛난다. 빛은 정보의 위계를 만든다.
3. **유전(流轉)** - 운세는 흐르는 것. 정적이지 않고 미세하게 움직인다.
4. **신비(神秘)** - 모든 것을 다 보여주지 않는다. 안개와 성운이 신비감을 더한다.
5. **격조(格調)** - 사주는 진지한 학문. 가벼운 판타지가 아닌 품격 있는 우주.

## 1. 컬러 시스템

### 1.1 배경 (Deep Space)

다크 모드 전용. 순수한 블랙(#000)이 아닌, 미세한 색조가 있는 딥 스페이스 컬러.

| 토큰명 | 값 | 용도 |
|--------|-----|------|
| `--space-void` | `#06080F` | 최하단 배경. 가장 깊은 우주. |
| `--space-deep` | `#0B0E1A` | 메인 페이지 배경 |
| `--space-mid` | `#111527` | 카드 내부, 섹션 배경 |
| `--space-surface` | `#181D35` | 상승된 표면 (모달, 드롭다운) |
| `--space-elevated` | `#1F2544` | 활성화된 표면, 호버 상태 |
| `--space-overlay` | `rgba(6, 8, 15, 0.85)` | 오버레이, 모달 뒤 딤 |

배경은 단색이 아니라 미세한 별 입자(star particle) 텍스처와 함께 사용한다. CSS에서 radial-gradient를 이용한 희미한 성운 효과를 배경에 깔 수 있다.

## 2. 타이포그래피

### 2.1 폰트 패밀리

| 역할 | 폰트 | 용도 |
|------|-------|------|
| **제목/강조** | Pretendard (Variable) | 모든 UI 제목, 버튼, 내비게이션. 깔끔한 한글 고딕. |
| **본문** | Pretendard (Variable) | 설명 텍스트, 입력 필드 |
| **한자 표기** | Noto Serif KR | 천간/지지 한자 (壬申, 己酉 등). 세리프체로 격조를 더함. |
| **숫자/영문** | Pretendard에 포함 | 날짜, 크레딧 숫자 등 |
| **장식/브랜드** | Gowun Batang | 브랜드 슬로건, 특별한 인용구에만 제한적 사용 |

Pretendard 선택 이유: 가변 폰트로 굵기 세밀 조절 가능, Apple SD 고딕과 비슷한 현대적 느낌, 무료, 한글/영문/숫자 모두 깔끔.

기존 Gowun Batang을 기본 폰트로 쓰던 것에서 전환한다. 조선 컨셉에서는 세리프체가 맞았지만, 우주/코스믹 컨셉에서는 산세리프(고딕)가 더 현대적이고 미래적인 느낌을 준다.

### 2.3 한자(漢字) 전용 스타일링

한자는 사주에서 핵심 정보를 담고 있으므로, 특별한 스타일링을 적용한다.

**사주 기둥 내 한자**
- 폰트: Noto Serif KR, weight 700
- 크기: 본문 대비 1.5~2배 (맥락에 따라)
- 색상: `--text-primary` 또는 해당 오행의 core 색상
- 효과: 미세한 text-shadow로 발광감 부여
  - 예: `text-shadow: 0 0 8px rgba(오행색, 0.4)`
- 배경: 해당 오행의 glow 색상으로 둥근 배경

**인라인 한자**
- 폰트: Noto Serif KR, weight 400
- 크기: 본문과 동일하되 약간 크게 (1.05em)
- 색상: `--text-hanja`

## 3. 컴포넌트 스타일 가이드

### 3.1 버튼

모든 버튼의 공통 속성:
- border-radius: 12px (부드러운 둥근 모서리)
- transition: all 200ms ease
- font-weight: 600
- 비활성 상태: opacity 0.4 + cursor not-allowed

#### Primary (CTA)
- 배경: `--gradient-cta` (보라 그래디언트)
- 텍스트: `--text-primary` (흰색 계열)
- 호버: 밝기 증가 + 미세한 위로 이동(-1px) + box-shadow 확대
- box-shadow: `0 4px 15px rgba(124, 92, 252, 0.3)`
- 호버 box-shadow: `0 6px 25px rgba(124, 92, 252, 0.45)`
- 활성(active): 살짝 눌리는 느낌 (scale 0.98)

#### Secondary
- 배경: `--space-surface`
- 테두리: 1px solid `--border-default`
- 텍스트: `--text-primary`
- 호버: 배경 `--space-elevated` + 테두리 밝아짐

#### Ghost
- 배경: transparent
- 텍스트: `--cta-secondary`
- 호버: 배경 `rgba(124, 92, 252, 0.08)`

#### Sun Button (해 소비 액션)
- 배경: `--gradient-sun`
- 텍스트: `--text-inverse` (어두운 색)
- 아이콘: 해 아이콘 + 소비량 표시 (예: "사주 풀이 받기 ☀ 1")
- box-shadow: `0 4px 15px rgba(255, 215, 0, 0.3)`
- 호버: 코로나 효과 확대 (box-shadow 범위 증가)
- 특수: 버튼 주변에 미세한 금빛 파티클 애니메이션 (선택적)

#### Moon Button (달 소비 액션)
- 배경: `--gradient-moon`
- 텍스트: `--text-inverse`
- 아이콘: 달 아이콘 + 소비량 표시 (예: "오늘 운세 보기 🌙 1")
- box-shadow: `0 4px 15px rgba(165, 180, 252, 0.25)`
- 호버: 후광 효과 확대

#### 크기 (모든 타입 공통)
| 크기 | padding | font-size | min-height |
|------|---------|-----------|------------|
| sm | 8px 16px | 14px | 36px |
| md | 10px 24px | 16px | 44px |
| lg | 14px 32px | 18px | 52px |

### 3.2 카드

기존 한지 카드에서 **코스믹 글래스모피즘**으로 전환.

#### 기본 카드 (Cosmic Glass)
- 배경: `rgba(17, 21, 39, 0.6)` (반투명 space-mid)
- backdrop-filter: `blur(12px)`
- 테두리: 1px solid `--border-subtle`
- border-radius: 16px
- box-shadow: `0 4px 24px rgba(0, 0, 0, 0.3)`
- 호버(인터랙티브 카드): 테두리가 `--gradient-card-border`로 변하면서 은은하게 빛남

#### 강조 카드 (Nebula Card)
- 기본 카드 + 배경에 미세한 성운 그래디언트 깔림
- 위쪽 또는 옆쪽에 오행색 기반 글로우 라인
- 예: 목 관련 카드는 상단에 `--wood-glow` 색의 1px 발광 라인

#### 서비스 카테고리 카드
- 기본 카드 기반
- 상단: 아이콘 영역 (48x48, 오행색 또는 보라색 글로우 배경에 아이콘)
- 중앙: 서비스명 (text-h3)
- 하단: 간단 설명 + 크레딧 표시 (달 or 해)
- 호버: 아이콘의 글로우가 확대되면서 카드 전체에 은은한 빛 번짐

#### 크레딧 패키지 카드
- 기본 카드 기반
- 선택된 상태: 테두리가 `--sun-core` 또는 `--moon-core`로 변경 + 글로우
- "인기" 배지: `--gradient-cta` 배경의 작은 pill
- "최고 가성비" 배지: `--gradient-sun` 배경의 작은 pill

### 3.3 입력 필드 (Input)

- 배경: `--space-mid`
- 테두리: 1px solid `--border-default`
- border-radius: 12px
- 텍스트: `--text-primary`
- placeholder: `--text-tertiary`
- padding: 12px 16px
- 포커스: 테두리 `--border-focus` + `--border-glow` (box-shadow)
- 에러: 테두리 `--status-error` + 에러 글로우
- 라벨: `--text-secondary`, overline 스타일, 필드 위 8px 간격

#### 셀렉트 / 드롭다운
- 입력 필드와 동일한 스타일
- 드롭다운 메뉴: `--space-surface` 배경, 글래스모피즘
- 선택된 항목: `--cta-primary` 배경 (투명도 낮춤)

#### 날짜/시간 입력 (사주 입력 전용)
- 커스텀 날짜 피커 (브라우저 기본 X)
- 년/월/일 각각 스크롤 드럼 방식 또는 숫자 입력
- 시간: 12시진 선택 UI (자시~해시, 각 시진을 원형 또는 그리드로 배치)
- "시간 모름" 옵션은 별도 체크박스/토글로 명확히 표시

### 3.4 모달

- 배경 딤: `--space-overlay`
- 모달 본체: `--space-surface` 배경, 글래스모피즘
- border-radius: 20px
- 테두리: 1px solid `--border-subtle`
- box-shadow: `0 24px 64px rgba(0, 0, 0, 0.5)`
- 최대 너비: 480px (모바일은 화면 대비 좌우 16px 마진)

- 퇴장: 페이드아웃 + 약간 아래로 (150ms)
- 닫기 버튼: 우상단 X, Ghost 버튼 스타일

### 3.5 내비게이션

#### 모바일 하단 탭바
- 배경: `rgba(11, 14, 26, 0.9)` + backdrop-blur(20px)
- 상단 테두리: 1px solid `--border-subtle`
- 높이: 64px + safe area bottom
- 탭 항목: 아이콘(24px) + 레이블(caption)
- 비활성: `--text-tertiary`
- 활성: `--cta-primary` + 아이콘 위에 작은 글로우 dot

탭 구성:
1. 홈 (별 아이콘)
2. 사주 (4개 기둥 아이콘)
3. 타로 (카드 아이콘)

#### 데스크톱 헤더

- 좌측: 로고 (브랜드명 또는 심볼)

- 우측: 크레딧 잔액 표시 + 프로필 아바타
- 하단 경계: 1px solid `--border-subtle`

### 3.6 크레딧 표시 (Sun/Moon Balance)

헤더 또는 마이페이지에 항상 표시되는 잔액 위젯.

**컴팩트 모드 (헤더)**
- 가로 배치: `☀ 12  🌙 47`
- 해: 금색 아이콘 + 금색 숫자
- 달: 은색 아이콘 + 은색 숫자
- 구분: 얇은 세로 디바이더
- 탭하면 크레딧 페이지로 이동

**확장 모드 (마이페이지, 크레딧 페이지)**
- 두 개의 작은 카드가 나란히
- 해 카드: `--sun-glow` 배경, 해 아이콘 크게, 잔액 숫자
- 달 카드: `--moon-glow` 배경, 달 아이콘 크게, 잔액 숫자
- 교환 버튼: "1☀ = 5🌙 교환" (중간에 배치)

### 3.7 오행 배지 (Five Nebula Badges)

오행을 표시하는 작은 pill 형태의 배지.

- 모양: pill (border-radius: 999px)
- 크기: padding 4px 12px
- 배경: 해당 오행의 `--{element}-glow` (투명도 있는 배경)
- 테두리: 1px solid 해당 오행의 `--{element}-core` (투명도 0.3)
- 텍스트: 해당 오행의 `--{element}-bright`
- 표시: "木 목" 또는 "木" (한자 + 한글, 또는 한자만)

사주 결과 화면에서 각 기둥의 오행을 배지로 표시. 오행 분포 차트에서도 사용.

### 3.8 로딩 상태

#### 풀페이지 로딩 (사주 계산 중)
- 딥 스페이스 배경
- 중앙: 별자리가 하나씩 이어지는 애니메이션
  - 4개의 빛점(사주 4기둥)이 순서대로 빛나면서 선으로 연결
  - "우주의 기운을 읽고 있습니다..." 텍스트
- 또는: 원형 궤도 위를 도는 빛의 점 (대운 궤도 메타포)

#### 인라인 로딩 (버튼 내부, 카드 내부)
- 작은 원형 스피너: 보라색 gradient 원이 회전
- 크기: 버튼 내부는 20px, 독립형은 32px

#### 스켈레톤
- `--space-mid` 배경에 `--space-elevated` 색의 shimmer 효과
- shimmer는 좌에서 우로 반짝이는 빛줄기 (성운 속 별빛 느낌)

## 4. 페이지별 레이아웃 컨셉

### 4.1 홈 (/)

**히어로 섹션: "우주의 기운을 드립니다"**
- 전체 뷰포트 높이 (100svh)
- 배경: `--gradient-milkyway` + 미세한 별 파티클 (CSS 또는 canvas)
  - 별들이 아주 천천히 반짝이는 애니메이션 (twinkle)
  - 중앙에 크고 희미한 원형 성운 (radial-gradient)
- 중앙 상단: 브랜드 로고/타이틀
- 중앙: "우주의 기운을 드립니다" (display 타이포, Gowun Batang)
  - 텍스트에 미세한 글로우 효과
- 아래: CTA 버튼 "내 사주 보기" (Primary)
- 하단: 스크롤 유도 화살표 (아래로 부드럽게 바운스)

**서비스 카테고리 그리드**
- 2x4 그리드 (모바일) 또는 4x2 (데스크톱)
- 각 카테고리 카드: 서비스 카테고리 카드 스타일
- 배치 순서: 오늘의 운세, 정통 사주, 애정운, 재물운, 신년운세, 토정비결, 내일 운세, 지정일 운세
- 카드 사이 간격: 12px
- 각 카드의 아이콘이 해당 서비스의 성격을 반영 (아래 아이코노그래피 참조)

**타로 배너**
- 전폭 카드, 내부에 그래디언트 배경 (보라+남색)
- 타로 카드 이미지가 3장 겹쳐서 기울어져 있는 비주얼
- "오늘의 타로 한 장" 제목 + CTA

**하단 CTA 섹션**
- "아직 사주를 모르시나요?" 문구
- "무료 사주 계산" 버튼

### 4.2 사주 입력 (/saju)

**상단: 단계 표시**
- 원형 궤도(타원) 위에 4개의 점: 성별 > 생년월일 > 시간 > 출생지
- 현재 단계의 점이 밝게 빛남, 완료된 단계는 빛나는 선으로 연결
- 또는 심플하게: 단계 번호 + 프로그레스 바 (보라 그래디언트)

**입력 폼**
- 한 화면에 한 단계씩 (스텝 바이 스텝) 또는 스크롤 형태
- 각 입력 그룹은 Nebula Card로 감싸기
- 성별: 음(☽)/양(☉) 아이콘이 있는 토글 또는 두 개 버튼
- 생년월일: 커스텀 날짜 피커
- 역법: 양력/음력/음력(윤달) 선택 - pill 형태 세그먼트 컨트롤
- 시간: 12시진 그리드 (2x6) + "시간 모름" 체크박스
  - 각 시진 셀: 시진명(子, 丑...) + 시간대(23:30-01:29)
  - 선택된 셀: 보라색 글로우 + 테두리
- 출생지: 검색 가능한 셀렉트 (시/도 > 시/군/구)

**하단: 제출 버튼**
- Sun Button 또는 Moon Button (해당 서비스의 크레딧 타입에 따라)
- 잔액 부족 시: 버튼 비활성 + "크레딧 충전하기" 링크

### 4.3 사주 결과 (/saju/result)

이 페이지가 시각적으로 가장 중요한 페이지. 사주의 모든 정보를 우주적으로 보여준다.

**4주 표시: 별의 기둥 (Star Pillars)**
- 화면 상단, 전체 폭
- 4개의 수직 기둥이 나란히 배치 (연주 | 월주 | 일주 | 시주)
- 각 기둥의 구조:
  ```
  ┌─────────┐
  │  연주    │ ← 기둥 레이블 (text-caption, 상단)
  │         │
  │   壬    │ ← 천간 한자 (크게, 발광, 해당 오행 색)
  │  임수    │ ← 한글 + 오행 (작게)
  │  ─────  │ ← 구분선 (미세한 빛 라인)
  │   申    │ ← 지지 한자 (크게, 발광, 해당 오행 색)
  │  신금    │ ← 한글 + 오행 (작게)
  │         │
  │ [木배지] │ ← 오행 배지
  └─────────┘
  ```
- 각 기둥은 `--gradient-pillar` 배경
- 기둥 사이에 미세한 빛의 선이 연결 (격국의 관계를 시각화)
- 시간 모름일 경우: 시주 기둥은 흐릿하게 + "?" 표시 + 성운 안개 효과

**격국 시각화: 별자리 패턴**
- 4개 기둥의 천간/지지를 점으로 표현 (총 8개 점)
- 이 점들을 관계에 따라 선으로 연결하면 고유한 별자리 패턴이 만들어짐
- 배경: 원형 차트 영역, 미세한 동심원 (궤도 느낌)
- 이 패턴이 사용자 고유의 "운명의 별자리"가 됨

**오행 분포: 성운 차트**
- 원형 차트 또는 막대 차트
- 각 오행의 비율을 해당 오행의 성운 그래디언트로 표현
- 중앙에 가장 강한 오행 표시
- 부족한 오행은 흐릿하게

**AI 해석: 우주의 메시지**
- 큰 Nebula Card 안에 해석 텍스트
- 상단에 아이콘과 "우주가 전하는 메시지" 등의 제목
- 본문은 body-lg 크기, 행간 넉넉히
- 핵심 키워드는 해당 오행 색으로 하이라이트

**용신/격국 정보**
- 카드 형태, 아이콘 + 짧은 설명
- 격국명은 h3, 설명은 body

### 4.4 타로 (/tarot)

## 5. 애니메이션 & 인터랙션

Framer Motion 기반. 성능을 위해 GPU 가속 속성(transform, opacity) 위주로 사용.

### 5.1 페이지 전환

**기본 전환**
- 진입: opacity 0 -> 1 + translateY(20px) -> 0 (300ms, ease-out)
- 퇴장: opacity 1 -> 0 + translateY(-10px) (200ms, ease-in)

**사주 결과 페이지 진입 (특별)**
- 먼저 배경이 어두워지면서 별이 하나씩 나타남 (500ms)
- 그 다음 4개의 기둥이 순서대로 아래에서 위로 올라옴 (각 200ms, stagger 100ms)
- 마지막으로 기둥들을 잇는 빛의 선이 그려짐 (300ms)
- 전체 시퀀스: 약 1.5초

### 5.2 카드 효과

**호버 (데스크톱)**
- 카드가 살짝 떠오름: translateY(-4px), 200ms
- 테두리가 빛남: border-color 변경 + box-shadow 확대
- 카드 내부에 미세한 빛 이동 (background-position 애니메이션)

**탭 (모바일)**
- 탭 시: scale(0.97) -> scale(1), 150ms
- 리플 효과 대신 글로우 효과 (탭 지점에서 빛이 퍼짐)

### 5.3 크레딧 소비/획득

**소비 시**
- 해 또는 달 아이콘이 버튼에서 빠져나와 위로 날아가면서 사라짐
- 잔액 숫자가 카운트다운 애니메이션 (이전 숫자 -> 새 숫자)
- 잔액 영역에 잠깐 글로우 반짝

**획득(충전) 시**
- 해 또는 달이 위에서 아래로 떨어지면서 잔액 영역에 도착
- 도착 시 글로우 펑 (burst) 효과
- 숫자 카운트업 애니메이션

### 5.4 로딩

**사주 계산 로딩 (풀페이지)**
- 4개의 빛점이 원형 궤도 위를 돈다
- 1번 점 -> 2번 점 -> 3번 점 -> 4번 점 순서로 밝아짐
- 모든 점이 밝아지면 선으로 연결되면서 완료
- 텍스트: "사주를 계산하고 있습니다..." -> "천간을 읽는 중..." -> "지지를 분석 중..." -> "오행의 조화를 살피는 중..."
- 각 단계별 텍스트가 fade-in/out으로 교체

**타로 셔플 로딩**
- 카드가 섞이는 애니메이션 (카드 3~5장이 교차하면서 위치 변경)

### 5.5 호버/탭 피드백

| 요소 | 데스크톱 호버 | 모바일 탭 |
|------|-------------|----------|
| 버튼 | 밝기 증가 + 미세한 떠오름 | 눌림(scale 0.97) + 글로우 |
| 카드 | 떠오름 + 테두리 발광 | 눌림 + 배경 밝아짐 |
| 링크 | 밑줄 + 색상 변경 | 색상 변경 |
| 아이콘 | 글로우 확대 | 글로우 펄스 1회 |
| 탭 항목 | - | 아이콘 바운스 + 글로우 |

### 5.6 미세 애니메이션 (Ambient)

페이지에 항상 존재하는 미세한 움직임. 성능에 부담 없는 수준.

- **별 반짝임**: 배경의 별 점들이 2~5초 주기로 밝기가 변함 (CSS animation, opacity)
- **성운 유동**: 배경 성운(gradient)이 아주 천천히 위치를 이동 (30초+ 주기, background-position)
- **궤도 라인**: 사주 결과 페이지에서 대운 궤도 라인 위의 점이 천천히 움직임

## 6. 아이코노그래피

아이콘 스타일: **라인 아이콘 (stroke 1.5~2px) + 핵심 부분만 fill**. Lucide 아이콘 베이스에 커스텀 아이콘 추가. 아이콘 크기 기본 24px, 카테고리 카드에서는 32~48px.

### 6.1 서비스 카테고리 아이콘 (8개)

| 서비스 | 아이콘 컨셉 | 설명 |
|--------|------------|------|
| 오늘의 운세 | 떠오르는 태양 + 별 | 수평선 위로 태양이 올라오고, 위에 작은 별 하나 |
| 정통 사주 | 4개의 빛 기둥 | 사주의 4기둥을 상징. 세로선 4개가 아래에서 빛남 |
| 애정운 | 쌍성(Binary Star) | 두 별이 가까이 공전하는 모습. 하트가 아닌 우주적 표현 |
| 재물운 | 황금 혜성 | 꼬리가 긴 혜성. 금색 포인트. 풍요를 상징 |
| 신년운세 | 새벽별 (금성) | 밝게 빛나는 큰 별 하나 + 작은 별들. 새로운 시작 |
| 토정비결 | 고서 + 별자리 | 펼쳐진 두루마리 위에 별자리 점 패턴 |
| 내일 운세 | 초승달 + 별 | 얇은 초승달과 그 옆의 별. 다가올 내일 |
| 지정일 운세 | 달력 + 궤도 | 달력 아이콘에 원형 궤도 선이 관통 |

### 6.2 크레딧 아이콘

**해 (Sun) 아이콘**
- 원형 중심체 + 방사형 광선 (8개 또는 12개)
- 색상: `--sun-core` ~ `--sun-corona` 그래디언트
- 크기별 디테일 차이:
  - 16px: 단순 원 + 짧은 광선
  - 24px: 원 + 광선 + 미세한 코로나
  - 48px: 원 + 광선 + 코로나 + 내부 패턴
- 애니메이션 가능: 광선이 미세하게 회전 (hover 시)

**달 (Moon) 아이콘**
- 그믐달(초승달) 형태. 왼쪽이 빛남.
- 색상: `--moon-core` ~ `--moon-halo` 그래디언트
- 크기별:
  - 16px: 단순 초승달
  - 24px: 초승달 + 1~2개 작은 별
  - 48px: 초승달 + 별들 + 미세한 후광
- 애니메이션 가능: 부드럽게 밝기 변화 (pulse, hover 시)

### 6.3 오행 아이콘

각 오행을 추상적인 우주 현상으로 표현. 구체적인 나무/불/흙 같은 지구적 이미지가 아닌, 우주적 이미지.

| 오행 | 아이콘 컨셉 | 설명 |
|------|------------|------|
| 木 (목) | 별 탄생 (Star Formation) | 성운에서 빛이 퍼져나오는 형태. 방사형이지만 유기적 곡선. |
| 火 (화) | 초신성 폭발 (Supernova) | 중심에서 강렬하게 빛이 퍼지는 형태. 뾰족한 광선. |
| 土 (토) | 행성 (Planet) | 안정적인 구체. 고리가 있어도 좋음 (토성 연상). |
| 金 (금) | 백색왜성 (White Dwarf) | 작지만 매우 밝은 점. 차갑고 날카로운 빛. 십자형 광채. |
| 水 (수) | 블랙홀 (Black Hole) | 원형 실루엣 + 주변의 강착원반(빛이 휘어지는 모습). 깊이감. |

### 6.4 내비게이션 아이콘

Lucide 아이콘 또는 유사 라인 아이콘 세트에서 선택.

| 탭 | 아이콘 | Lucide 후보 |
|----|--------|------------|
| 홈 | 별 | `Star` 또는 `Sparkles` |
| 사주 | 4기둥 | 커스텀 (세로선 4개) |
| 타로 | 카드 | `Layers` 또는 커스텀 |
| 크레딧 | 해+달 | 커스텀 (해와 달이 합쳐진 형태) |
| 마이 | 사람 | `User` 또는 `CircleUser` |

## 7. 반응형 & 크로스 플랫폼

### 7.1 브레이크포인트

| 이름 | 범위 | 대상 |
|------|------|------|
| mobile | ~639px | 모바일 (주력) |
| tablet | 640~1023px | 태블릿 |
| desktop | 1024px~ | 데스크톱 |

모바일 우선 설계. 사주 앱은 모바일 사용이 80% 이상일 것으로 예상.

### 7.2 React Native WebView 대응

- 하단 safe area 대응 (env(safe-area-inset-bottom))
- 스크롤 바운스 제어
- 터치 타겟 최소 44x44px
- backdrop-filter 미지원 기기 대비: fallback으로 불투명 배경색 제공
- 과도한 CSS animation 지양: WebView에서 성능 저하 가능

### 7.3 컨텐츠 최대 폭

- 본문 영역: max-width 640px (모바일 최적화 기준)
- 넓은 레이아웃: max-width 1024px (데스크톱 그리드)
- 전폭 요소: 히어로, 사주 기둥 표시 등은 제한 없음

## 9. 접근성 (Accessibility)

다크 모드 전용이므로 특히 주의해야 할 사항:

- 텍스트 대비: WCAG AA 기준 최소 4.5:1 (본문), 3:1 (대형 텍스트)
  - `--text-primary` (#EDEEF2) on `--space-deep` (#0B0E1A) = 약 15:1 (충분)
  - `--text-secondary` (#9BA1B7) on `--space-deep` (#0B0E1A) = 약 7:1 (충분)
  - `--text-tertiary` (#5E6480) on `--space-deep` (#0B0E1A) = 약 3.5:1 (대형 텍스트만)
- 오행 색상만으로 정보 전달하지 않기: 항상 텍스트 레이블 병행
- 포커스 링: 모든 인터랙티브 요소에 `--border-focus` + `--border-glow` 적용
- 애니메이션: `prefers-reduced-motion` 미디어 쿼리로 애니메이션 축소/제거 대응
- 글로우/반짝임: 간질 유발 가능성 - 깜빡임 빈도 3Hz 이하로 제한
